# PHÂN TÍCH NGUYÊN NHÂN CRASH KHI ẤN "THAY ĐỔI" (CHỌN FOLDER) TRÊN WINDOWS

Báo cáo phân tích chi tiết nguyên nhân lỗi crash ứng dụng MusicYT khi người dùng nhấp vào nút **"Thay đổi"** để mở hộp thoại chọn thư mục (`SelectSavePath`) trên hệ điều hành Windows.

---

## 🔍 1. Triệu Chứng Lâm Sàng
Khi người dùng chuyển sang tab bất kỳ và click vào nút **"Thay đổi"** (ở phần chọn đường dẫn lưu trữ), ứng dụng đột ngột biến mất (crash hoàn toàn) mà không hiển thị thông báo lỗi trên UI, không sinh log lỗi thông thường trong DevTools console.

---

## 🛠️ 2. Phân Tích Nguyên Nhân Gốc (Root Causes)

Hàm xử lý khi click **"Thay đổi"** ở Frontend (`App.tsx` & `UniversalDownloader.tsx`):
```typescript
const handleSelectPath = async () => {
    try {
        const path = await SelectSavePath(); // Gọi sang Go Backend binding
        if (path) setSaveLocation(path);
    } catch (err) {
        console.error(err);
    }
};
```

Hàm `SelectSavePath` xử lý ở Go Backend (`app.go`):
```go
func (a *App) SelectSavePath() (string, error) {
	path, err := wailsRuntime.OpenDirectoryDialog(a.ctx, wailsRuntime.OpenDialogOptions{
		Title: "Chọn thư mục lưu video",
	})
	if err != nil {
		return "", err
	}
	return path, nil
}
```

Có **3 nguyên nhân chính** dẫn đến vụ sập (crash) này trên Windows:

### 🔴 Nguyên Nhân 1: Lỗi Dereference Con Trỏ Nil (`a.ctx` bị Nil)
- **Cơ chế hoạt động:** Hàm `wailsRuntime.OpenDirectoryDialog` bắt buộc phải nhận vào một `context.Context` hợp lệ (`a.ctx`) để tìm kiếm Handle của cửa sổ cha (Parent Window Handle) và hiển thị hộp thoại đè lên trên.
- **Vấn đề:** Nếu vì bất kỳ lý do gì mà ứng dụng gọi `SelectSavePath` khi `a.ctx` chưa được khởi tạo (bằng `nil`), hàm `OpenDirectoryDialog` sẽ cố gắng truy cập vùng nhớ của `a.ctx` dẫn đến lỗi truy cập vùng nhớ trái phép (**Nil Pointer Dereference**). 
- **Tại sao xảy ra?** 
  - Trong các bài chạy thử nghiệm (Unit Test) hoặc các môi trường giả lập, struct `App` được tạo bằng `NewApp()` nhưng không chạy qua vòng đời `wails.Run` nên hàm `startup(ctx)` không được kích hoạt, làm cho `a.ctx` bằng `nil`.
  - Có sự bất đồng bộ (Race Condition) nhẹ giữa luồng khởi tạo Wails GUI và luồng React Frontend. Khi React render lần đầu tiên, nếu nó gửi yêu cầu quá sớm trước khi `OnStartup` hoàn tất việc lưu `a.ctx`, lỗi này sẽ phát sinh.

### 🔴 Nguyên Nhân 2: Xung Đột Threading COM trên Windows (STA vs MTA)
- **Cơ chế hoạt động:** Hộp thoại chọn thư mục/file native trên Windows (thông qua Shell API `IFileOpenDialog`) yêu cầu thread gọi nó phải chạy dưới mô hình luồng đơn **STA (Single-Threaded Apartment)**.
- **Vấn đề:** 
  - Wails nhận cuộc gọi IPC từ Frontend và chạy các hàm binding Go trên các goroutine riêng biệt (được ánh xạ sang các luồng hệ điều hành MTA - Multi-Threaded Apartment).
  - Mặc dù Wails có cơ chế tự động chuyển lệnh gọi Dialog về GUI Thread chính (STA), nhưng trong một số phiên bản Wails v2 hoặc khi chạy trên một số cấu hình Windows đặc thù, cơ chế đồng bộ này bị thất bại. Khi một thread MTA cố gắng mở hộp thoại STA, Windows sẽ ném ra lỗi **Access Violation (0xc0000005)** làm sập ngay toàn bộ tiến trình ứng dụng.

### 🔴 Nguyên Nhân 3: Xung đột Go Version (Go 1.23+ & Wails API)
- **Vấn đề:** Gần đây, các bản cập nhật Go mới (từ 1.23 trở lên) thay đổi cách quản lý và tối ưu hóa luồng hệ điều hành trên Windows (Assembly Syscalls). Sự thay đổi này tạo ra xung đột nghiêm trọng với thư viện liên kết động WebView2 và các API Dialog của Windows mà Wails đang đóng gói, gây crash ngẫu nhiên khi gọi các hộp thoại Native Dialog nếu không đồng bộ luồng cực kỳ chặt chẽ.

## 🛡️ 3. Phác Đồ Điều Trị & Giải Pháp Khắc Phục (Solutions)

Để giải quyết triệt để vấn đề này, chúng ta cần thực hiện các bước tối ưu hóa sau tại Go Backend:

### ✅ Giải pháp 1: Vá lỗi chặn Nil Pointer Dereference & Tự động khôi phục
Thêm bước kiểm tra an toàn (Safe Check) đối với `a.ctx` trước khi gọi hộp thoại. Nếu `a.ctx` bị nil, ta ghi log/trả về lỗi thay vì để ứng dụng bị crash. Đồng thời dùng cơ chế `recover()` để hứng tất cả panic.

```go
func (a *App) SelectSavePath() (string, error) {
	// Kiểm tra an toàn bảo vệ tiến trình
	if a.ctx == nil {
		return "", fmt.Errorf("không thể mở hộp thoại: Context chưa được khởi tạo")
	}

	// Hứng panic nếu Windows COM bị lỗi đột ngột
	defer func() {
		if r := recover(); r != nil {
			fmt.Printf("Hệ thống khôi phục từ sự cố Dialog: %v\n", r)
		}
	}()

	path, err := wailsRuntime.OpenDirectoryDialog(a.ctx, wailsRuntime.OpenDialogOptions{
		Title: "Chọn thư mục lưu video",
	})
	if err != nil {
		return "", err
	}
	return path, nil
}
```

### ✅ Giải pháp 2: Đồng bộ hóa phiên bản & Cấu hình môi trường
- Đảm bảo phiên bản Wails CLI và thư viện Wails trong `go.mod` được cập nhật đồng bộ lên phiên bản mới nhất:
  ```powershell
  go get github.com/wailsapp/wails/v2@latest
  ```
- Nếu đang chạy Go 1.23+ và gặp lỗi crash liên tục, cân nhắc chạy ứng dụng với Go 1.22.x ổn định hơn cho Wails v2 để tránh xung đột luồng Windows COM.
